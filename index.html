Setup Instructions

Follow these steps exactly before trying to log in or upload to Google Drive.

Open setup steps
  1. Go to Google Cloud Console and create a project.
  2. Enable the Google Drive API for that project.
  3. Open OAuth consent screen and configure app name, support email, and test users if the app is in testing mode.
  4. Create OAuth 2.0 credentials with type Web application.
  5. Add your deployed site to Authorized JavaScript origins, for example `https://yourdomain.com` or your GitHub Pages URL.
  6. Copy the generated Client ID and paste it into the Google OAuth Client ID field in this app.
  7. Host this file on HTTPS. `file://` local open will not work for Google Drive OAuth.
  8. Click Connect Google Drive, allow Drive access, then upload files.
  9. The app creates a public permission, uses the Drive file viewer URL for QR generation, and includes the Drive resource key when available.
  10. Delete is permanent. The confirmation dialog intentionally warns before continuing.
Required scopes
openid profile email https://www.googleapis.com/auth/drive
